# Prod_Test_Tool_v3

## 개요
`Prod_Test_Tool_v3`는 IP 카메라 검사 현장에서 사용하는 통합 운영 도구입니다.

이 프로그램은 단순한 테스트 보조 UI가 아니라,  
**현장에서 실제로 불편했던 작업들을 한 프로그램 안에서 바로 처리할 수 있도록 개선한 생산용 툴**입니다.

기존에는 현장에서 주로 아래 도구들을 사용했습니다.

- **True Manager**
- **Test Tool**

하지만 실제 운영에서는 아래와 같은 한계가 있었습니다.

- 기능이 제한적임
- TTA 같은 **보안 펌웨어 환경에서는 사용이 제한됨**
- 웹에서 따로 테스트하거나 웹에서 펌웨어를 올려야 해서 불편함
- 다중 제어가 어려움
- 여러 제품의 화면을 띄울 수는 있어도 **제어가 안 됨**
- TTA 환경에서는 **화면 확인 자체가 어려움**

즉, 생산/검사 현장에서는  
장비마다 다른 도구를 따로 쓰거나, 웹과 프로그램을 오가면서 작업해야 했고,  
특히 보안 펌웨어 장비나 다중 장비 운영에서는 불편이 매우 컸습니다.

`Prod_Test_Tool_v3`는 바로 이 문제를 개선하기 위해 만든 프로그램입니다.

---

## 이 프로그램이 만들어진 배경

이 프로젝트와 관련된 내부 프로그램은 아래 두 가지가 있었습니다.

- **as_test_tool**
  - AS 용도로 직접 만든 자체 프로그램
- **prod_test_tool_v2**
  - 생산용으로 직접 만든 자체 프로그램

즉, 이 프로젝트는 외부 샘플을 그대로 가져온 것이 아니라,  
**직접 운영하면서 쌓인 경험을 바탕으로 기존 자체 프로그램들을 발전시킨 결과물**입니다.

정리하면:

- `as_test_tool`에서는  
  실제 현장에서 검증된 인증 방식, 제품 정보 조회 방식, 제어 방식, 표시 방식을 계승했고
- `prod_test_tool_v2`에서는  
  화면 분리, 상태 관리, worker 흐름, 구조적 분리 방식을 참고했습니다
- 그리고 현장에서 사용하던 `True Manager / Truen Test Tool`의 한계를 보완하는 방향으로
  **생산용 통합 프로그램**으로 재구성했습니다

---

## 무엇이 개선되었는가

이 프로그램은 아래를 목표로 만들어졌습니다.

### 1. 웹과 별도 프로그램을 오가지 않도록 통합
기존에는:
- 어떤 작업은 프로그램에서 하고
- 어떤 작업은 웹에서 하고
- 어떤 작업은 장비별로 따로 해야 했습니다

이제는 아래 작업을 **한 프로그램 안에서** 처리할 수 있습니다.

- IP 변경
- Reset
- 비밀번호 변경
- 보안 펌웨어( TTA 포함 / Security 3.0 ) 비밀번호 처리
- 접속
- 장비 정보 조회
- 장비 제어
- 장비 다중 제어
- 장비 다중 비디오 출력
- 장비 다중 펌웨어 업로드

즉, 생산 직원 입장에서는  
**복잡한 절차 없이 실행해서 바로 작업할 수 있는 통합 도구**로 보는 것이 맞습니다.

### 2. 보안 펌웨어 환경 대응
기존 도구는 TTA 같은 보안 펌웨어 환경에서 제약이 컸습니다.

이 프로그램은 다음을 포함해 보안 펌웨어 환경을 고려했습니다.

- `none / basic / digest` 인증 처리
- 기본 비밀번호 상태 판별
- 비밀번호 변경 로직
- Security 3.0 bootstrap
- TTA 환경 접속 처리
- 보안 환경 장비의 정보 조회 / 제어 / 펌웨어 흐름 대응

### 3. 다중 작업 강화
기존에는:
- 여러 화면은 띄울 수 있어도 제어가 어려웠고
- 다중 제어가 제한적이었으며
- 펌웨어 작업도 대량 운영 관점에서 불편했습니다

이 프로그램은 아래를 개선했습니다.

- 체크 장비 기준 다중 제어
- 장비 다중 비디오 출력
- 10대 단위 비디오 페이지 운영
- 장비 다중 펌웨어 업로드
- 개별 장비 실패가 전체 작업을 멈추지 않는 구조

### 4. 현장 사용성을 우선한 UI
이 프로그램은 개발자가 아니라 **생산 직원이 쓰는 프로그램**이라는 점을 가장 중요하게 봤습니다.

그래서 UI는 아래 기준으로 구성했습니다.

- 한눈에 이해 가능
- 버튼명 / 상태명 / 섹션명이 직관적
- 복잡한 기술 용어 최소화
- 무엇을 눌러야 하는지 바로 보이도록 구성
- 기능이 많아도 화면이 복잡해 보이지 않게 분리

## 이 프로그램으로 할 수 있는 일

`Prod_Test_Tool_v3`는 아래 작업을 **한 프로그램 안에서** 처리할 수 있습니다.

### 장비 준비 / 네트워크 관리
- IP Discovery
- 장비 추가
- IP 변경
- Reset

### 연결 / 인증
- 장비 접속
- 비밀번호 변경
- 보안 펌웨어 장비 접속
- TTA 환경 장비 접속

### 장비 조회
- 장비 정보 불러오기
- 상태 정보 확인
- 모델 / 버전 / 보드 / 저장장치 / 온도 / 센서 / 알람 등 확인

### 장비 제어
- PT / Zoom / Focus
- TDN / ICR
- Reboot / Factory Reset
- Model Name / Extra ID / RTC Sync
- Air Wiper
- Video Setting
- Minimum Focus Length
- 485 Sensor
- Shock Sensor
- Audio / Video Input 관련 제어

### 다중 작업
- 체크 장비 다중 제어
- 장비 다중 비디오 출력
- 장비 다중 펌웨어 업로드

## 주요 기능

### 1. Discovery / Add Device / SetIP / Reset
- UDP Discovery
- 장비 목록 추가
- 선택 장비 SetIP
- Reset 기능
- 메인 화면과 자연스럽게 이어지는 준비 흐름 제공

### 2. Connect / Info / Status
- 장비 연결
- 인증 처리
- 제품 정보 조회
- 상태 정보 조회
- raw key 대신 사용자 친화적 표시명 제공

### 3. Control
- 단일 제어 / 다중 제어 지원
- 생산용 제어 기능을 섹션 단위로 정리
- 일반 직원도 기능 위치를 바로 찾을 수 있도록 구성

### 4. Video Mode
- 별도 Video 창
- 10대 단위 페이지 운영
- checked / selected / focused 기준 대상 해석
- 더블클릭 fullscreen
- 타일 단위 오류 격리

### 5. Firmware Mode
- 메인과 분리된 별도 창
- `.tus` 파일 기준 펌웨어 작업
- 선택 → 파일 지정 → 시작 → 진행 상태 확인 → 실패 장비 재시도
- reconnect polling 기반 완료 판정
- 일부 장비 실패가 전체 작업을 멈추지 않도록 설계

## 설계 원칙

이 프로젝트는 아래 원칙을 중요하게 봅니다.

### 1. 전체 크래시 금지
- 장비 1대 실패가 전체 프로그램에 영향을 주면 안 됨
- 영상 1개 타일 실패가 전체 영상 창을 죽이면 안 됨
- 펌웨어 1개 job 실패가 전체 batch를 멈추면 안 됨

### 2. Step 단위 개발
- 한 번에 전체를 만들지 않음
- 기능별로 분리해서 구현
- 각 단계마다 실행 / 테스트 / 통과 후 다음 단계로 이동

### 3. UI는 생산 직원 기준
- 버튼명은 짧고 직관적으로
- 기술 용어 최소화
- 자주 쓰는 흐름이 앞에 보이도록 구성

### 4. 기존 구조를 함부로 흔들지 않음
- 검증된 backend 구조는 유지
- 기능은 별도 창 / 별도 controller / 별도 supervisor로 확장
- 운영 중 발견되는 이슈를 점진적으로 보강
